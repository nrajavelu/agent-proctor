# 🔧 Violation Timing Fix - Technical Documentation

## Problem Identified
The proctoring system was triggering **video-related violations** before the video processing system was properly initialized, causing false positives for:
- "Face not clearly visible in camera feed" 
- "Multiple faces detected in frame"
- Other camera/video-dependent violations

This happened because violations were being generated by simulation **immediately** when a session started, before camera access was granted and video analysis was ready.

## Root Cause Analysis
1. **No initialization delay** - Violations started immediately on session start
2. **No video readiness check** - System didn't wait for camera/video stream  
3. **No differentiation** between behavioral vs video-dependent violations
4. **No warmup period** - No grace time for system initialization

## Solution Implemented

### 1. **Video System Initialization** 
```typescript
const initializeVideoSystem = () => {
  console.log('📹 Initializing video system...');
  
  // Simulate video system initialization
  videoInitTimeoutRef.current = setTimeout(() => {
    setIsVideoReady(true);
    console.log('✅ Video system ready for violation detection');
  }, 2000); // 2-second video init time
};
```

### 2. **Warmup Period Protection**
```typescript
const isInWarmupPeriod = () => {
  if (!sessionStartTime) return true;
  return (Date.now() - sessionStartTime) < 5000; // 5-second warmup period
};
```

### 3. **Violation Type Classification**
```typescript
const violationTypes = [
  // Non-video violations (can trigger immediately)
  { type: 'tab-switch', requiresVideo: false },
  { type: 'unauthorized-device', requiresVideo: false },
  { type: 'audio-anomaly', requiresVideo: false },
  
  // Video-dependent violations (only after video ready + warmup)
  { type: 'face-not-visible', requiresVideo: true },
  { type: 'multiple-faces', requiresVideo: true },
  { type: 'suspicious-behavior', requiresVideo: true }
];
```

### 4. **Smart Violation Filtering**
```typescript
violationTypes.forEach(vType => {
  // Skip video violations if video not ready or in warmup
  if (vType.requiresVideo && (!isVideoReady || isWarmup)) {
    return;
  }
  
  // Proceed with violation logic...
});
```

### 5. **UI Status Indicators**
- **Video System Status:** Shows "Initializing...", "Video Loading", "Video Ready"
- **Warmup Period Indicator:** Clear feedback during initialization
- **Test Button States:** Video violation test buttons disabled during warmup

## Timing Sequence

| Time | Behavioral Violations | Video Violations | Status |
|------|---------------------|------------------|---------|
| 0s | ✅ Active | ❌ Blocked | "Initializing..." |
| 2s | ✅ Active | ❌ Blocked | "Video Loading" |
| 5s | ✅ Active | ✅ Active | "Video Ready" |
